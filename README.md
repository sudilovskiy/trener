# Trener

Android workout tracking app with progress analytics and BLE weight integration.

---

## Key Features

- Workout session tracking (multi-day split)
- Exercise progress analytics (current vs previous)
- Body weight tracking (manual + BLE scale)
- Workout history with detailed sessions
- Backup & restore support

---

## Tech Stack

- Kotlin
- Jetpack Compose
- Room (SQLite)
- Android BLE
- Health Connect

## Description (RU)





# Тренер

`Тренер` — Android-приложение для учета домашних тренировок и контроля веса.

## Что умеет приложение

- Вести тренировки по 3 тренировочным дням.
- Запускать тренировку с таймером и сохранять подходы по упражнениям.
- Показывать историю тренировок, детали занятия и редактирование сохраненных данных.
- Строить график веса по диапазону дат.
- Добавлять вес вручную.
- Читать вес из `Health Connect`, если на устройстве уже выданы нужные разрешения.
- Подключаться к BLE-весам и сохранять стабильное измерение в историю веса.
- Создавать резервную копию базы и восстанавливать данные из архива.

## Требования

- Android 7.0+.
- Для BLE-весов нужен Bluetooth и системные разрешения на работу с BLE.
- Для импорта из `Health Connect` на устройстве должен быть установлен `Health Connect`, а приложению должен быть разрешен доступ к данным веса.

## Основные сценарии

### 1. Тренировка

1. На главном экране выберите `День 1`, `День 2` или `День 3`.
2. Нажмите `Начать тренировку`.
3. Открывайте упражнения из списка и заполняйте подходы.
4. Для каждого подхода можно указать текущее значение и заметку.
5. Нажмите `Завершить` у подхода, чтобы сохранить его.
6. После завершения тренировки нажмите `Завершить тренировку`.

Что важно:

- Пока есть активная тренировка, приложение не даст начать другую.
- Если выйти с главного экрана во время активной тренировки, приложение предложит завершить и сохранить её.
- Для упражнения показывается прогресс и сравнение текущей суммы повторений с предыдущей тренировкой.

### 2. История тренировок

Откройте `История тренировок` с главного экрана.

Здесь можно:

- посмотреть список всех сохраненных тренировок;
- открыть детали конкретной тренировки;
- отредактировать сохраненную тренировку;
- удалить отдельную тренировку;
- удалить все тренировки;
- создать резервную копию;
- восстановить данные из резервной копии.

### 3. Работа с весом

На главном экране доступны 3 сценария:

- `Добавить вес` — ручное сохранение текущего веса;
- `Управление весом` — удаление одной записи, диапазона записей или всей истории веса;
- `Весы MI` — подключение к BLE-весам и импорт веса напрямую с устройства.

График веса поддерживает:

- диапазоны `7 дней`, `30 дней`, `Все`, `Свой диапазон`;
- ручной выбор диапазона по датам;
- просмотр данных по всей сохраненной истории веса.

### 4. Импорт веса из Health Connect

Приложение пытается синхронизировать вес автоматически при открытии главного экрана и при возврате на него.

Импорт сработает только если:

- `Health Connect` установлен на устройстве;
- в `Health Connect` уже есть записи веса;
- приложению уже выдан доступ на чтение веса.

Если хотя бы одно из условий не выполнено, приложение продолжит работу без автосинхронизации.

### 5. Подключение BLE-весов

Экран `Весы MI` используется для поиска и подключения к совместимым BLE-весам.

Как пользоваться:

1. Откройте `Весы MI`.
2. Разрешите Bluetooth и BLE-разрешения, если система их запросит.
3. Встаньте на весы для обнаружения устройства.
4. Выберите найденные весы или подключитесь к ранее сохраненному основному устройству.
5. Дождитесь стабильного значения.
6. После сохранения веса экран автоматически закроется.

Если весы не поддерживаются или значение не удалось стабилизировать, приложение покажет статус ошибки или отсутствия данных.

## Резервное копирование и восстановление

Резервная копия создается из раздела `История тренировок`.

- `Создать резервную копию` — сохраняет архив с локальной базой данных.
- `Восстановить из резервной копии` — заменяет текущие локальные данные содержимым архива.

Перед восстановлением приложение покажет предупреждение: текущие данные будут удалены.

## Ограничения

- Приложение хранит данные локально на устройстве.
- Автоматическая синхронизация веса из `Health Connect` не заменяет ручной ввод и BLE-импорт, а дополняет их.
- Поддержка BLE зависит от модели весов и от того, публикуют ли они совместимые характеристики веса.

## Обслуживание репозитория

В проекте есть утилиты для контроля локальных кэшей и временных Android/Gradle-артефактов, которые не должны раздувать репозиторий.

Основные команды:

- `tools\repo-status.bat` — показать состояние репозитория и подозрительные локальные каталоги.
- `tools\repo-fix.bat` — привести проект к безопасному базовому состоянию.
- `tools\repo-clean-build.bat` — дополнительно очистить `build` и `app\build`.
- `tools\repo-gradle.bat <task>` — запустить `gradlew` с безопасными путями для локальных кэшей.

Краткая инструкция находится в [tools/README.md](tools/README.md).
