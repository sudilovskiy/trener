# Repo Maintenance Tools

Эти скрипты нужны для Android/Gradle-проектов, у которых локальные кэши или временные артефакты начинают складываться внутрь репозитория и раздувают его размер.

Что они решают:

- находят подозрительные папки вроде `.gradle-user`, `.gradle-home`, `.userhome`, `.android-sdk`;
- помогают не смешивать код проекта с локальными кэшами Gradle и Android;
- быстро приводят проект в безопасное состояние после переноса, неудачного запуска или экспериментов с окружением;
- позволяют запускать `gradlew` с нормальными путями к Gradle cache и Android SDK.

Когда использовать:

- проект внезапно начал весить сотни мегабайт или гигабайты;
- после запуска Android Studio в корне появились лишние `.gradle-*`, `.android-*`, `.userhome`;
- нужно перенести этот набор в другой Android-проект и быстро привести его к такому же состоянию;
- нужно запустить Gradle из терминала, не рискуя снова сложить кэши в папку проекта.

Команды:

- `tools\repo-status.bat`
  Показывает текущее состояние: переменные окружения и подозрительные каталоги в репозитории.

- `tools\repo-fix.bat`
  Базовая безопасная команда. Проверяет проект, дополняет `.gitignore` и удаляет нештатные дублирующиеся локальные кэши внутри репозитория.

- `tools\repo-clean-build.bat`
  Дополнительно удаляет `build` и `app\build`. Используй, если нужно полностью очистить артефакты сборки.

- `tools\repo-gradle.bat <gradle task>`
  Запускает `gradlew` с безопасными путями для `GRADLE_USER_HOME`, `ANDROID_USER_HOME`, `ANDROID_SDK_ROOT`.

Примеры:

```bat
tools\repo-status.bat
tools\repo-fix.bat
tools\repo-clean-build.bat
tools\repo-gradle.bat assembleDebug
tools\repo-gradle.bat test
```

Как перенести в другой проект:

1. Скопируй папки `scripts` и `tools` в корень другого проекта.
2. Запусти `tools\repo-fix.bat`.
3. При необходимости используй `tools\repo-status.bat` для проверки и `tools\repo-gradle.bat` для безопасного запуска Gradle.

Что важно:

- обычные `app\build` и `.gradle` после сборки Android Studio могут появляться снова, это нормально;
- скрипты в первую очередь борются не с обычной сборкой, а с нештатными дублирующимися локальными кэшами внутри репозитория;
- Android Studio не будет автоматически запускать эти скрипты сама, их нужно запускать вручную или встроить в свой процесс работы.
